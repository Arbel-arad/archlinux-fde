firejail fixes for gocryptfs

by default firejail will try to scan the home directory for profile.local files.

i'm not sure why it's done, but it feels like a risk and won't work anyway in our case.

the better option is to put your profile.local files in /etc/firejail/ as if a profile is found there no search will be required in the home directory.

do NOT run firecfg without verifying all profiles work correctly or some apps might not open.

you can test it, and then disable the profiles with firecfg --clean
