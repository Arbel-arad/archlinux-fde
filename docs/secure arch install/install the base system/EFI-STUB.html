EFI-STUB

efibootmgr --create --disk /dev/nvme0n1 --part 1 --label "Arch Linux" --loader /EFI/BOOT/

limitations of EFI-STUB:

  • /dev/EFI-DRIVE has to be a hardware drive (no /dev/mapper)
  • --part is the GPT partition number, they start from 1, not 0. use fdisk /dev/EFI-DRIVE to list the partitions.

initramfs unified-kernel-image contains the kernel parameters, therefore making them immutable when used in combination with the TPM and secure-boot to validate the signed image.

the kernel parameters can be provided from an encrypted directory as they are only read by the initramfs generator, while the system is booted.

the unencrypted ESP should only contain signed verified initrd files. (no configs that can be edited externally).
